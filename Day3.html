Basic DBMS Terminologies

Today we will be discussing about the some basic terminologies of database management system. We will briefly discuss them and get key insights about them.


The basic terminologies in DBMS are:

>Data

Data refers to information that is collected, stored, and analyzed. It can take various forms, such as numbers, text, images, audio, and video




>Information

Information is processed, organized, or structured data that is meaningful and useful for decision-making, understanding, or communication.



>Database

A structured collection of data that is stored and accessed electronically.Databases are managed using DBMS software, which ensures data integrity and security.


>Database Management System

A Database Management System (DBMS) is software that allows users to create, manage, and manipulate databases.


>Table

In a Database Management System (DBMS), a table is a structured set of data organized in rows and columns.


>Keys

Keys in a Database Management System (DBMS) are attributes that are used to uniquely identify records within a table and to establish relationships between tables.


>Normalization

Normalization in DBMS is the process of organizing the columns(attributes) and tables(relations) of a database to reduce data redundancy and improve data integrity.



>Domain

In a Database Management System (DBMS), a domain refers to the set of permissible values that a column can hold.


>SQL

Structured query language (SQL) is a programming language for storing and processing information in a relational database.


>Xampp

XAMPP is a free and open-source cross-platform web server solution stack package developed by Apache Friends, consisting mainly of the Apache HTTP Server.
