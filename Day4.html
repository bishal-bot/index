Keys in DBMS

Today we will be discussing about the some important keys of database management system. We will briefly discuss them and get key insights about them.


Definition

In a Database Management System (DBMS), keys are attributes or sets of attributes that help in uniquely identifying a record in a table. A key is also used

when we want to establish relationships between the different columns and tables of a relational database.


The important keys in DBMS are:

>> Primary Key

> Definition:

A primary key is a column or a combination of columns that uniquely identifies a record in a table.

> Features/Uses:

1. It must contain unique values.

2. It cannot contain NULL values.


>> Candidate Key

> Definition:

A candidate key is a set of columns that can uniquely identify a record in a table.

> Features/Uses:

1. A table can have multiple candidate keys.

2.One of the candidate keys is chosen as the primary key.


>> Super Key

> Definition:

A super key is a set of one or more columns that can uniquely identify a record in a table.

> Features/Uses:

1. All candidate keys are super keys, but not all super keys are candidate keys.

2. A super key may contain extra attributes that are not necessary for unique identification.


>> Alternate Key

> Definition:

An alternate key is any candidate key that is not chosen as the primary key.

> Features/Uses:

1. It serves as a secondary means of identifying a record uniquely.


>> Composite Key

> Definition:

A composite key is a primary key that consists of two or more columns.

> Features/Uses:

1. It is used when a single column is not sufficient to uniquely identify a record.


>> Foreign Key

> Definition:

A foreign key is a column or a set of columns in one table that references the primary key in another table.

> Features/Uses:

1. It is used to establish and enforce a link between the data in the two tables.


>> Secondary Key

> Definition:

A secondary key is a non-unique key used for indexing and retrieval purposes.

> Features/Uses:

1. It is not used to uniquely identify records but to speed up query processing.
